Task 1: Scan Your Local Network for Open Ports

This report presents a security risk assessment of open ports identified across devices in a local network range (192.168.1.0/24) using Nmap scans.

Ports ,Services and Security Risk Based Overview

Nmap scan report for 192.168.1.1

Port Protocol Service Security Risk
21 TCP FTP Transmits data in plain text. Vulnerable to sniffing and brute-force attacks. Replace with SFTP or disable.
22 TCP SSH Brute-force attack target. Should use key-based authentication and monitoring.
23 TCP Telnet Targeted by brute-force and dictionary attacks. If weak passwords or default credentials are used, can be compromised.
53 TCP DNS Can allow DNS zone transfers if misconfigured, leaking internal network details.
80 TCP HTTP No encryption; allows interception of data (MITM). Can be exploited via vulnerable web apps. Redirect to HTTPS.
139 TCP NetBIOS-SSN Vulnerable to NetBIOS spoofing and session hijacking. Often exploited by malware.
161 TCP SNMP Exposes device info if default credentials are used. Should use SNMPv3 with encryption.
443 TCP HTTPS Encrypted, but vulnerable if using outdated TLS versions or misconfigured certificates.
445 TCP SMB Used in Windows file sharing. Vulnerable to major exploits like EternalBlue.
3517 TCP 802.11 IAPP Rarely used. May be exposed by misconfigured wireless devices. (e.g., AP controller).
8082 TCP BlackICE Alerts / Custom App Legacy or unknown service. Could expose web admin panels or logs.

Nmap scan report for 192.168.1.7

Port Protocol Common Service Security Risk
6881 TCP BitTorrent (P2P)
  • Malware Distribution: BitTorrent traffic may include pirated or infected content.
  • Legal Liability: Illegal file sharing can result in copyright violations or regulatory fines.
  • Bandwidth Saturation: Can consume excessive bandwidth, impacting performance of critical systems.
  • Network Obfuscation: P2P traffic can bypass traditional proxy or monitoring systems.
  • DDoS Involvement: Nodes can be abused for Distributed Denial-of-Service attacks.
  • Botnet Recruitment: Malware using P2P protocols (like Peerbot or ZeroAccess) may use this port for C2 communication.
  • Hidden Services: Port 6881 can be used by malicious services masked as BitTorrent traffic.
  • Firewall Evasion: Because it's often allowed for home use, attackers may exploit it to exfiltrate data.

Nmap scan report for 192.168.1.12

Port Protocol Common Service Security Risk
8009 TCP Apache JServ Protocol (AJP)
  • Ghostcat (CVE-2020-1938): Remote file inclusion vulnerability allowing reading server files.
  • Unauthenticated access: AJP assumes trusted internal use, risking unauthorized access if exposed.
  • File read/write: Potential for remote file access or uploads if misconfigured.
  • Internal request forwarding: Can bypass firewalls or security controls.
  • Potential for Remote Code Execution: When combined with other vulnerabilities.
  • Lack of encryption: Traffic is unencrypted by default, risking interception.
  • Often overlooked: Rarely monitored, making it a stealthy attack vector.
